This repository is designed to help contributors learn **real-world open-source workflows**.  
Please follow the guidelines below to ensure a smooth and efficient review process.

---

## How to Contribute

Follow these steps to get started:

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your changes  
   **Example:** `docs-improve-readme`
4. Make your changes
5. Commit your changes with a clear message
6. Push the branch to your fork
7. Open a Pull Request (PR)

---

## Writing Guidelines

When contributing documentation or content:

- Use clear and simple language
- Keep explanations concise and well-structured
- Use proper Markdown formatting
- Avoid unnecessary repetition
- Ensure grammatical correctness

---

## Commit Message Guidelines

### Good commit messages
- `docs: improve README clarity`
- `docs: add contribution examples`

### Avoid commit messages like
- `update`
- `fix stuff`
- `changes made`

Commit messages should clearly describe **what** was changed.

---

## Pull Request Guidelines

- One PR should address **one issue only**
- Link the related issue in the PR description
- Follow the PR template (if available)
- Be open to feedback and requested revisions

> Maintainers may request changes before merging.

---

## What Not to Do

- Do not submit auto-generated or AI-dumped content
- Do not open PRs without reading the issue description
- Do not spam or duplicate contributions

---

## Getting Help

If you are unsure about anything, feel free to comment on the issue **before** starting work.  
Maintainers and contributors are happy to help.
