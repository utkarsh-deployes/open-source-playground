# Contributing to Open Source Playground

Thank you for your interest in contributing.

This repository is designed to help contributors learn real open-source workflows. Please follow the guidelines below to ensure a smooth review process.

---

## How to Contribute

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your changes  
   Example: `docs-improve-readme`
4. Make your changes
5. Commit with a clear message
6. Push to your fork
7. Open a Pull Request

---

## Writing Guidelines

- Use clear, simple language
- Keep explanations concise and structured
- Use proper markdown formatting
- Avoid unnecessary repetition
- Ensure grammatical correctness

---

## Commit Message Guidelines

Good commit messages:
- `docs: improve README clarity`
- `docs: add contribution examples`

Avoid:
- `update`
- `fix stuff`
- `changes made`

---

## Pull Request Guidelines

- One PR should address one issue
- Link the related issue in the PR description
- Follow the PR template
- Be open to feedback and revisions

Maintainers may request changes before merging.

---

## What Not to Do

- Do not submit auto-generated or AI-dumped content
- Do not open PRs without reading the issue
- Do not spam or duplicate contributions

---

## Getting Help

If you are unsure about anything, comment on the issue before starting work.
